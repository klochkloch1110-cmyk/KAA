# RLS policies AVL 84

Основная миграция содержит базовый набор RLS-политик. Этот файл фиксирует смысловую модель доступа.

## Admin

Админ видит и изменяет все данные.

## Operator

Оператор на будущее. Может читать операционные данные и ограниченно редактировать заявки/справочники. В первом релизе роль можно создать, но интерфейс оператора не выделять отдельно.

## Driver

Водитель может:
- читать свой профиль;
- читать назначенные ему заявки;
- читать свои рейсы;
- создавать свои рейсы по назначенным заявкам;
- читать и обновлять свои смены;
- читать назначенную машину в рабочем контексте;
- читать документы своих рейсов/смен;
- создавать документы к своим рейсам/сменам.

Водитель не может:
- читать чужие рейсы;
- читать чужие смены;
- читать чужие документы;
- видеть чужие начисления;
- создавать и редактировать заявки;
- редактировать справочники;
- создавать машины и пользователей.

## Заявки и управленческая ставка

`orders.admin_rate_per_unit` - управленческое поле. Клиентский UI водителя не должен запрашивать и отображать это поле.

Важно: RLS в PostgreSQL ограничивает строки, а не отдельные колонки. Поэтому для водительского чтения добавлено представление `public.driver_orders_safe` без `admin_rate_per_unit`.

Правило для клиента:
- админский экран заявок может читать `public.orders` и видеть `admin_rate_per_unit`;
- водительские экраны должны читать `public.driver_orders_safe` или RPC на его основе;
- прямой `select` из `public.orders` для водительского клиента использовать нельзя.
