# Seed data AVL 84

Seed-файлы нужны для локальной и dev-среды.

## Dev seed

Файл `001_dev_seed.sql` создает базовые справочники, машины, заявки и рабочий чат. Пользователи приложения создаются только если соответствующие UUID уже существуют в `auth.users`.

Перед запуском на реальном Supabase-проекте:

1. создайте пользователей в Supabase Auth;
2. замените UUID в блоке `dev_seed_ids` на реальные `auth.users.id`;
3. примените seed через SQL Editor или CLI;
4. не коммитьте реальные UUID production-пользователей, пароли и токены.

## Что стоит добавить первым

- 1 админ;
- 2 водителя;
- 2 машины;
- 2 заказчика;
- 3 организации;
- 3 материала;
- 3 локации;
- 2 тестовые заявки;
- 1 открытая смена;
- 2 тестовых рейса.

Важно: реальные пароли и секреты не хранить в seed-файлах репозитория.
