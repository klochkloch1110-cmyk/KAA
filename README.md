# AVL 84

Рабочая папка проекта внутренней системы управления перевозками для компании AVL 84.

## Структура проекта

- `ideya.md` - исходное описание идеи, процессов и пожеланий к приложению.
- `app_blueprint_v1.md` - проектный каркас v1: модули, роли, данные, roadmap и общий подход.
- `app/` - Flutter-каркас приложения для Android водителя и web-кабинета руководителя.
- `docs/` - основная проектная документация: экраны, статусы, ER-модель, архитектура, UI/UX-ТЗ.
- `supabase/` - стартовая серверная часть: SQL-миграции, RLS, storage, seed и заготовки Edge Functions.
- `prototypes/` - визуальные и интерактивные прототипы.

## Текущий прототип

Прототип из Figma Make лежит отдельно:

- `prototypes/figma-make/`

Это React/Vite-проект, который можно запускать отдельно от основной документации. Он нужен как визуальный и сценарный референс, а не как финальная архитектура приложения.

Инструкция по запуску: `prototypes/figma-make/README.md`.

## Рекомендуемый порядок работы

1. Использовать `docs/06_development_plan_v1.md` как главный план ближайшего этапа.
2. Поднять Supabase-проект и применить миграции из `supabase/migrations/`.
3. Проверить RLS-политики на ролях админа и водителя.
4. Установить Flutter SDK и выполнить команды из `app/README.md`.
5. Реализовать первый сквозной сценарий: заявка -> рейс -> фото ТТН -> смена -> выгрузка.

## Текущий инженерный фокус

Первый рабочий релиз 0.1 должен закрыть операционное ядро:

```text
Админ создает заявку
-> назначает водителя и машину
-> водитель видит заявку
-> водитель отправляет рейсовый отчет с фото ТТН
-> водитель закрывает смену
-> админ видит рейсы, смены и документы
-> админ выгружает таблицу по рейсам
```

OCR, чат, push-уведомления, зарплата, расходы, ТО и AI-агент заложены в архитектуру, но не блокируют первый рабочий релиз.
